## 1. 攻击距离公式 (Brotato 风格加法)

**设计**: 玩家 attackRange 与武器 attackRange **相加** (Brotato 风味), 非乘数.

**公式**:
```js
const weaponRange = (wp.attackRange || 60) + (p.attackRange || 0);
```

**11 角色 attackRange 加成** (`csv/characters.csv` + `src/data/characters.json` + `src/data/data-bundle.js`):

| 角色 | attackRange | 流派 |
|------|-------------|------|
| swordsman 剑客 | +50 | 平衡 |
| gunslinger 枪手 | +120 | 远程主 |
| fire_mage 火焰法师 | +100 | 远程 AOE |
| archer 弓手 | +150 | 超远距离 |
| mech 机械 | -30 | 近战坦克 |
| assassin 刺客 | -20 | 突进 |
| medic 医生 | 0 | 平衡 |
| paladin 圣骑士 | +30 | 团队 |
| engineer 工程师 | +60 | 工具 |
| berserker 狂战士 | -50 | 贴身爆发 |
| dragon_knight 龙骑士 | +80 | 范围 |

**cd 平衡** (Brotato 风格: 范围大 → cd 增加):
```js
const rangeNorm = Math.min(1, weaponRange / 200); // 0~200 归一
const cdScale = 1 + rangeNorm * 1.0; // 范围 200px ≈ +100% cd
const finalCd = baseCd * cdScale;
```

**回滚理由**: 之前用减法 (weapon 减 player) 在多数情况下数值会过小 (< 30), 玩家攻击距离没有差异化.

## 3. 命中检测 (角色 + 锥形, Brotato 风格)

**设计原则**: Brotato 风格 — 判定中心 = **角色位置**, 武器 sprite 是**视觉**动画锚点. 锥形朝 `attackAngle` 方向, 范围 = `weaponRange`.

**算法** (`_tickMeleeHitDetection`):
```js
// Brotato 加法: 武器 + 角色
const weaponRange = (wp.attackRange || 60) + (p.attackRange || 0);
const aa = wp._attackAngle;
// 锥形: thrust 60° / sweep 180° (朝 attackAngle 方向)
const cone = (wp._attackBehavior === 'melee_sweep') ? Math.PI / 2 : Math.PI / 3;

// 怪: 1) 距角色 ≤ weaponRange,  2) 在锥形内
const dx = e.x - p.x, dy = e.y - p.y;
const dist = Math.sqrt(dx * dx + dy * dy);
const eR = e.radius || 14;
if (dist > weaponRange + eR) continue;          // 远怪不命中
const ang = Math.atan2(dy, dx);
let diff = ang - aa;
diff = ((diff + Math.PI) % (2 * Math.PI)) - Math.PI;
if (Math.abs(diff) > cone) continue;             // 反方向怪不命中
```

**满足 3 个需求**:
- ✓ **长枪聚集怪**: 长枪 weaponRange=160, 怪距角色 30 ≤ 160, 锥形朝怪 (0±30°) 在锥内 → 命中
- ✓ **方向感**: 攻击"上" → 锥形朝上, "下"怪 (90°) 在锥外 (-90°±30°) → 不命中
- ✓ **距离感**: 怪 300px > 160 → 不命中

**修复历史**:
1. ~~sprite 圆心 + 32 半径~~ → 长枪轨道 128px 外打不到聚集怪
2. ~~角色 + 大圆~~ → 无方向感
3. ~~sprite 中心 + 大圆 + 60° 锥形~~ → 锥形方向跟怪相对 sprite 中心方向相反, 永远过不了
4. ~~sprite 中心 + 线段碰撞 (不 clamp t)~~ → 反方向怪 + 远怪都命中 (问题 1 + 问题 2)
5. ✓ **当前**: 角色 + 锥形 → 3 个需求都满足

**sprite 用途** (不参与判定):
- 视觉动画: thrust 摆动 (轨道 56px → maxDist), sweep 弧形扫
- 击退方向: sprite 中心 → 怪 (沿用 `kdx, kdy`)

**长枪场景**:
- pike 长枪 `attackRange: 160`, sprite 在 56~168px 轨道
- 怪聚集角色 (30, 0): 距角色 30 ≤ 160, atan2(0,30)=0, 锥形 0±30° ✓ 命中
- 怪在下 (0, 30): atan2(30,0)=90°, 锥形朝右 (0±30°) → 不在锥内 → 不命中 ✓
- 远怪 (300, 0): 距 300 > 160 → 不命中 ✓

## 4. 攻击行为选择 (thrust vs sweep)

**3 条规则** (`_fireWeapon`):

### 4.1 `melee_sweep` 武器 (等离子刀/能量斧): 周围 ≥3 怪才用 sweep
```js
if (actualBehavior === 'melee_sweep') {
    let nearCount = 0;
    for (let j = 0; j < enemies.length; j++) {
        if (enemies[j].alive && 距角色 < meleeRange) nearCount++;
    }
    actualBehavior = (nearCount >= 3) ? 'melee_sweep' : 'melee_thrust';
}
```

**设计理由**: sweep 是"群怪时大招", 默认走 thrust, 群怪时升级为 sweep.

### 4.2 `melee` 武器 (无明确): 按距离选
- `tag === 'lance'` 永远 `melee_thrust` (长枪/三叉戟/骑枪)
- 否则: 距目标 < `meleeRange * 0.45` → `melee_sweep`; 否则 `melee_thrust`

### 4.3 `melee_thrust` 武器 (武士刀/能量剑): 永远 thrust

**行为对照表**:

| 武器 | def.behavior | 行为 |
|------|--------------|------|
| plasma 等离子刀 | melee_sweep | 周围<3怪 → thrust, ≥3怪 → sweep |
| axe 能量斧 | melee_sweep | 周围<3怪 → thrust, ≥3怪 → sweep |
| sword 能量剑 | melee_thrust | 永远 thrust (60° 锥 + pierce:3) |
| katana 武士刀 | melee_thrust | 永远 thrust (60° 锥 + pierce:3) |
| lance 长枪 | melee + tag:lance | 永远 thrust (60° 锥 + pierce:4) |
| trident 三叉戟 | melee + tag:lance | 永远 thrust (60° 锥 + pierce:6) |

### 4.4 锥形参数
- thrust: **60° 锥** (朝 attackAngle ±30°)
- sweep: **180° 弧** (朝 attackAngle ±90°)

### 4.5 pierce 处理
- pierce = 0: 一次只打 1 怪 (用 `_meleeHit_<uid>` 标记)
- pierce > 0: 累计命中, 超过 pierce 后 `_meleeHit_<uid>` 标记防重扣
- 清理: 攻击结束后清 `_meleeHit_*` 标记 + `_hitCount`

## 5. 武器购买 (槽位未满开新, 满了才升级)

**设计**: 槽位是稀缺资源, **优先开槽**而非合并升级, 玩家有更多"同时持有"选择权.

**规则** (`buyItem`):
```js
if (usedSlots < player.weaponSlots) {
    // 1) 槽位未满 → 加新武器 (即使有同 id 也不合并)
    player.weapons.push({ id, level: 1, quality: 'T1' });
    action = 'bought';
} else {
    // 2) 槽位已满 → 找同 id 升级; 没同 id 失败
    const existing = player.weapons.findIndex(w => w.id === shopItem.id);
    if (existing !== -1) {
        player.weapons[existing].level++;
        action = 'merged';
    } else {
        return null; // 槽位满 + 没同 id
    }
}
```

**回滚理由**: 之前买重复 1 级 → 直接升 2 级, 玩家没有"双持同 id 1 级"的选择.

**测试**:
- S34: 槽位未满时同 ID 不合并 (加新槽位)
- S34b: 槽位已满时同 ID 合并升级

## 6. Animator 动画系统

**设计**: 通用 Animator 抽象层, idle/attack/death 三状态, 渲染层只读 `getTransform`.

**接口契约**:
```js
const transform = Animator.getTransform(animator);
// → { scale: number, rotation: number, frame: number }
```

**状态机**:
- **idle**: 4%/500ms 正常呼吸 (怪: 13%/130ms 快速呼吸)
- **attack**: 13%/130ms 快速震动
- **death**: easeOutQuad 渐缩 0, 600ms

**实现** (`src/engine/animator.js`):
- STATES: idle/attack/death
- `create({ phase })`: 创建实例, phase 错相位避免齐整
- `setState(state)`: 切换状态
- `update(dt)`: 推进 time
- `getTransform(animator)`: 返回纯数据, 渲染层负责应用

**错相位** (怪):
```js
enemy.animator = Animator.create({ phase: enemy._uid * 0.7 });
```

**死亡**:
```js
// 玩家 takeDamage 死亡
Animator.setState(p.animator, 'death');
// scale = baseScale * (1 - (1-t)²), 600ms 渐缩到 0
```

**双环境兼容**:
- 浏览器: 末尾 `globalThis/window/global.Animator = Animator` (无 export)
- vitest: `var _Animator = globalThis.Animator || null` 兜底
- ESM 顶层 var 仍 module-scope

## 10. 浏览器兼容约束

### 10.1 `<script>` 共享全局 lexical scope
- 顶层 `const` / `let` 跨文件**冲突** → `SyntaxError: Identifier already declared`
- **必须用 `var`** (允许重名, 后者覆盖)
- ESM 顶层 var 也是 module-scope (vitest Node ESM 兼容)

### 10.2 `<script>` 不支持 `import` / `export`
- 必须删 `export { X }`, 改用 `globalThis.X = X` (末尾)

### 10.3 顶层 `var` 重复声明
- 浏览器: 静默覆盖 (后者胜)
- Node ESM: 允许 (module-scope)
- 测试 vitest: 允许
