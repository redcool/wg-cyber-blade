# CYBER BLADE · 重构计划 v2

> 基于 docs2 设计文档，从"可跑"到"好玩"的全面重构
> 最后更新：2026-05-29

---

## 一、为什么重构

MVP 验证了核心循环可跑通，但**缺乏深度**：

| 问题 | 表现 |
|------|------|
| 道具无趣 | 大部分是纯加数值，没有"这局成了！"的爽感 |
| Build 不成立 | 羁绊简单、无流派概念、无跨系统联动 |
| 角色雷同 | 仅属性差异，没有规则修改和代价 |
| Boss 无感 | 数值放大版精英，没有机制挑战 |
| 敌人单调 | AI 类型少，不会克制不同 Build |
| 奖励枯燥 | 无稀有度、无保底、无流派偏向 |

## 二、重构目标

> **核心原则：让游戏好玩。行为 > 数值，机制 > 属性。**

### 2.1 设计哲学

- **Tag 系统**是一切联动的中心
- **道具** 30%+ 必须是机制型（OnHit/OnKill/OnCrit/Burn...）
- **角色**有明确的优势 + 代价 + 玩法变化
- **敌人**按"克制 Build"设计，行为比数值重要
- **Boss**多阶段、有机制、强制移动
- **奖励**有稀有度分级 + 保底 + 流派偏向
- **不让无敌 Build 出现**

### 2.2 新核心架构

```
角色 (被动技能 + 代价)
  ↓
武器 ←── Tag系统(7种标签) ──→ 道具
  ↓                              ↓
伤害公式(4层)              触发机制(OnHit/OnKill/OnCrit...)
  ↓                              ↓
敌人(行为>数值) ← 波次(预算制生成)
  ↓
Boss(多阶段+机制) → 奖励(商店/升级/宝箱 稀有度+保底+流派偏向)
```

## 三、关键系统设计

### 3.1 伤害公式（从单层到四层）

**旧公式：** `damage = baseDmg × damageMult`

**新公式：** `FinalDamage = (Base + Flat) × (1 + Percent) × Crit × Special`

| 层 | 含义 | 示例 |
|----|------|------|
| **B** (Base) | 武器基础伤害 | 手枪 10 |
| **F** (Flat) | 固定加成 | 近战+6, 远程+5 |
| **P** (Percent) | 全局百分比 | Damage +50% → 1.5 |
| **C** (Crit) | 暴击期望 | `1 + CritChance × (CritDmg - 1)` |
| **S** (Special) | 特殊倍率 | 对燃烧敌人 +20%, 满血 +30% |

**AttackSpeed 不进单次伤害公式**，避免"攻速×伤害"双爆炸。DPS = FinalDamage × AttackSpeed。

### 3.2 属性系统（6 大类）

| 类别 | 属性 | 说明 |
|------|------|------|
| **生存** | MaxHP, HPRegen, LifeSteal, Armor, Dodge, HealingModifier | 活下去 |
| **输出** | DamagePercent, MeleeDamage, RangedDamage, ElementalDamage, AttackSpeed, CritChance, CritDamage, Engineering | 打出去 |
| **机动** | Speed, Range, Knockback | 走位 |
| **经济** | Luck, Harvesting, XPGain, MaterialGain | 滚雪球 |
| **特殊** | ExplosionDamage, ExplosionSize, BurningSpread, TurretStats, ProjectilePierce, Bounce | 流派专属 |
| **限制** | 武器类型限制, 武器数量限制, 属性锁定, 成本变化 | 角色代价 |

### 3.3 Tag 系统（流派的核心）

7 个核心标签，驱动所有系统联动：

```
Melee | Ranged | Fire | Explosive | Crit | Tech | Economy
```

**联动规则：**
- 同 Tag 武器数量 → 触发 synergy 加成
- 商店/宝箱/升级卡 → 流派偏向（+20% 出现概率）
- 道具的 OnHit/OnKill 效果 → 按 Tag 分类触发
- 角色被动技能 → 绑定 Tag

### 3.4 道具系统（4 层结构）

```
Item = Stats(属性) + Triggers(触发) + Tags(标签) + Rarity(稀有度)
```

#### 稀有度

| 稀有度 | 概率 | 特点 |
|--------|------|------|
| Common | 60% | 小幅数值 |
| Rare | 25% | 数值 + 小机制 |
| Epic | 10% | 强机制 |
| Legendary | 5% | 改变玩法（核心件） |

#### 触发器类型

| 触发器 | 说明 | 示例 |
|--------|------|------|
| OnHit | 攻击命中时 | 20% 概率点燃 |
| OnKill | 击杀时 | 爆炸 |
| OnCrit | 暴击时 | 释放闪电链 |
| OnDamageTaken | 受伤时 | 反弹伤害 |
| OnDodge | 闪避时 | 回血 |
| PerSecond | 每秒 | 1 HP/s 回复 |
| OnLowHP | 低血量时 | 伤害翻倍 |

### 3.5 6 大核心流派

| 流派 | 核心 | 关键道具 |
|------|------|---------|
| 🔥 Fire (燃烧) | DOT + 扩散 | 点燃概率, 燃烧传播, 对燃烧增伤 |
| 💥 Explosive (爆炸) | AOE 清屏 | 爆炸范围, 连锁爆炸, 击杀触发爆炸 |
| 🎯 Crit (暴击) | 高爆发 | 暴击率, 暴伤, 暴击触发效果 |
| 🩸 Sustain (吸血) | 打不死 | 吸血, 受伤回血, 低血强化 |
| 🤖 Tech (工程) | 自动战斗 | 炮塔增强, 数量增加, 工程缩放 |
| 💰 Economy (经济) | 滚雪球 | 金币→伤害, 收获成长, 贪欲之戒 |

**质变节点：** 每个流派有"核心件"——拿到后玩法质变（如"燃烧可以传播"、"暴击触发连锁"）。

### 3.6 角色系统（优势 + 代价 + 被动）

| 角色 | 优势 | 代价 | 被动技能 |
|------|------|------|---------|
| Default (默认) | 均衡 | 无 | 无 |
| GlassCannon (玻璃大炮) | +50% 伤害 | -5 护甲 | — |
| Tank (坦克) | +10 护甲, +50 HP | -20% 伤害, -5 速度 | — |
| Berserker (狂战士) | +30% 伤害, +0.5 攻速 | HP Regen=0 | 低血时 +30% 伤害 |
| Engineer (工程师) | Engineering+10 | -10% 伤害, 攻速-0.1 | 工程物 +50% 伤害 |
| Pyromancer (火法) | +20% 伤害, 攻击附加燃烧 | — | 火焰 +20% 伤害 |
| Hunter (猎人) | +10% 暴击, +10 射程 | — | 远程 +5 伤害 |
| Merchant (商人) | +20% 金币获取 | -10% 伤害 | 每 50 金币 +5% 伤害 |
| Assassin (刺客) | +20% 暴击, +15% 闪避 | -2 护甲 | 暴击时 +50% 伤害 |
| Elementalist (元素师) | +10 魔法伤害 | — | 元素 +15% 伤害, 燃烧传播 |

### 3.7 被动技能系统

数据驱动的触发系统：

```
PassiveID | TriggerType | Condition | EffectType | EffectValue | Target | Tags
```

| 触发器 | 说明 |
|--------|------|
| Passive | 永久生效 |
| OnHit | 攻击命中 |
| OnKill | 击杀 |
| OnCrit | 暴击 |
| OnDamageTaken | 受伤 |
| PerSecond | 每秒 |
| OnLowHP | 低血量触发 |

### 3.8 敌人系统（行为 > 数值）

#### 行为类型（7 种）

| 行为 | 说明 | 克制 |
|------|------|------|
| Chaser | 追击玩家 | 测试基础输出 |
| Runner | 高速低血 | 克制低机动 |
| Tank | 高血高甲低速 | 克制 AOE 不足 |
| Shooter | 保持距离射击 | 强迫移动 |
| Bomber | 接近自爆 | 压迫空间 |
| Swarm | 极低血极多数量 | 测试清怪能力 |
| Summoner | 召唤小怪 | 优先击杀 |

#### 特殊机制敌人（5 种）

| 类型 | 机制 |
|------|------|
| Splitter (分裂) | 死亡后分裂 |
| Shielded (护盾) | 必须先破盾 |
| Leech (吸血) | 攻击回复 |
| Reflect (反伤) | 反弹伤害 |
| Freezer (减速) | 减速/眩晕 |

#### Build 克制关系

| 玩家流派 | 被克制方 |
|----------|---------|
| AOE 流 | Tank |
| 单体流 | Swarm |
| 站桩流 | Bomber |
| 工程流 | 高速怪 |

### 3.9 波次系统（Budget 制）

```
每波预算 = WaveNumber × 10
敌人按权重从池中随机选取
while budget > 0: spawn(enemy), budget -= enemy.cost
```

#### 难度曲线

| 波次 | 特点 |
|------|------|
| 1~3 | 教学，仅 Chaser |
| 4~6 | 引入 Runner, Shooter |
| 7~10 | 混合敌人+Tank |
| 11~15 | 特殊敌人+Bomber |
| 16+ | 全面混合+高压 |

#### 刷新模式（SpawnPattern）

- 随机分布（默认）
- 环形包围
- 定点刷怪
- 波浪推进

### 3.10 Boss 系统（多阶段）

每个 Boss 3 阶段，每阶段改变行为和技能：

```
Phase 1 (100%~70% HP): 基础攻击 + 召唤小怪
Phase 2 (70%~30% HP): 新技能 + 攻击速度提升
Phase 3 (<30% HP):     终极技能 + 高压模式
```

**Boss 设计原则：**
- 多阶段
- 明确机制（不是纯数值放大）
- 强制玩家移动
- 奖励必定 Epic/Legendary

### 3.11 奖励系统（稀有度 + 保底 + 流派偏向）

#### 商店

- 每次刷新 3~5 件商品
- 稀有度分布：Common 60% / Rare 25% / Epic 10% / Legendary 5%
- 保底：每 3 次必出 Rare，每 10 次必出 Epic，每 20 次必出 Legendary
- 流派偏向：当前 Build 的 Tag 商品 +20% 出现概率

#### 升级抽卡（Level-Up）

- 每次升级 3~5 张卡
- 类型：属性加成 / 武器增强 / 特殊机制
- 流派偏向：当前 Build 标签 +20~30% 权重

#### 宝箱

| 宝箱类型 | Common | Rare | Epic | Legendary |
|----------|--------|------|------|-----------|
| 普通 | 70% | 25% | 5% | 0% |
| 精英 | 40% | 35% | 20% | 5% |
| 传奇 | 10% | 20% | 30% | 40% |

## 四、技术决策

### 保持纯 JS

- 不引入 TypeScript 或构建工具（Vite/Webpack）
- 保持"双击 index.html 就能跑"的零门槛
- 全局命名空间模式（`const GameEngine = {}`, `const PlayerSystem = {}`...）
- 建议引入 ESLint 做代码风格约束（不影响运行）

### 数据格式：JSON 替代 CSV

- 当前 CSV 解析代码复杂且脆弱（手写 splitCSVLine 到处复制）
- JSON 原生支持，`fetch('data/xxx.json').then(r => r.json())`
- 嵌套结构更自然（武器含 scaling 子对象、道具含 effects 数组）
