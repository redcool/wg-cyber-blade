## 三、核心模块详解

### 3.1 主引擎 (main.js)

**GameEngine** — 游戏循环与状态机：

```
状态: menu → playing → shopping → playing → ... → gameover
               ↑            ↑
               └── levelup ──┘
```

核心方法：
- `init()` — 初始化所有子系统，启动游戏循环
- `_loop()` — requestAnimationFrame 主循环，dt 限幅 0.05s
- `_updatePlaying(dt)` — 更新玩家/敌人/子弹/粒子/关卡
- `_checkBulletCollisions()` — 碰撞检测（玩家子弹→敌人，敌人子弹→玩家）
- `_dropMaterials()` — 击杀掉落金币（10%概率2枚，90%概率1枚）
- `_updateMaterials(dt, player)` — 金币吸引 + 生命周期管理
- `_handleEnemyKill()` — 共享击杀处理（统计/经验/吸血/掉落/宝箱/日志/粒子/音效）
- `closeShop()` — 关闭商店 → 升级或下一关

**GameWorld** — 世界常量：
- `width/height`: 3000×3000
- `materials[]`: 掉落地面的金币（含吸引动画、生命周期30秒）

### 3.2 玩家系统 (player.js)

**PlayerSystem** — 单例对象管理：
- `create(x, y)` — 创建玩家，应用角色属性，初始化武器参数
- `update(dt, enemies)` — 移动(键盘) + 击退 + 自动攻击 + 回血 + 拾取
- `_attackWithAllWeapons(target)` — 所有装备武器同时开火
- `_fireWeapon(weaponId, params, target)` — 派发到 8 种武器行为
- `_pickupMaterials()` — （已迁移到 main.js 的 `_updateMaterials()`）
- `takeDamage(rawDmg)` — 闪避判定 + 护甲减伤 + 无敌帧 → 播放受击音效
- `addXP(amount)` — 经验积累 + 升级触发 → 播放升级音效

武器行为方法：
| 方法 | 行为 | 音效 |
|------|------|------|
| `_fireBullet()` / `_fireLaser()` | 标准/激光子弹 | `shoot` |
| `_fireSpread()` / `_fireShock()` | 散射/电击 | `shoot` |
| `_fireExplode()` | 爆炸范围 | `shoot` |
| `_fireFrost()` | 冰霜减速 | `shoot` |
| `_fireHoming()` | 跟踪弹 | `shoot` |
| `_fireSpray()` | 锥形喷射 | `shoot` |
| `_fireMeleeSweep()` / `_fireMeleeThrust()` | 近战横扫/突刺 | `enemy_hit` |

### 3.3 子弹系统 (bullet.js)

**BulletSystem** — 对象池模式管理子弹对象：

```
bullets[]    ← 活跃子弹
pool[]       ← 回收待重用
```

对象池减少了 GC 压力。子弹属性包括：
- 基础：坐标/速度/伤害/穿透/生命周期
- 武器来源：weaponId（用于渲染区分）
- 特殊：chainCount/chainRange/splashRadius/homingStrength/slowAmount

`update(dt)` — 每帧对所有子弹更新位置、跟踪逻辑、爆炸判定、越界/过期清除

### 3.4 敌人系统 (enemy.js)

**EnemySystem** — 6 种敌人类型定义 + AI 行为：

- `chase` 行为：追逐玩家 + 碰撞伤害 + 击退玩家
- `ranged` 行为：保持距离 + 远程射击

`takeDamage(enemy, damage)` — 伤害处理 + 击退 + 死亡返回 -1

### 3.5 关卡系统 (wave.js)

**WaveSystem** — 无限关卡生成：

- `currentLevel` 递增，每关 30 秒
- 敌人类型池随关卡解锁（basic → fast → tank → ranged → elite → boss）
- 每 5 关生成精英/BOSS
- 难度缩放：`1 + (level-1) * 0.15`
- 生成间隔从 1.5s → 最少 0.4s
- 敌人数：`4 + floor(level * 1.5)`

### 3.6 商店系统 (shop.js)

**ShopSystem** — 武器+道具双轨商店：

- `allWeapons[10]` — 所有武器定义（行为、槽位、价格、属性修正）
- `allItems[12]` — 所有道具定义（纯增益效果函数）
- `generateItems()` — 从已解锁池随机生成 3~4 武器 + 3~4 道具
- `buyItem(index)` — 购买逻辑（槽位检查/替换、属性修正叠加、记录解锁）
- `_applyWeaponMods()` — 叠加武器的 +/- 属性修正到玩家

### 3.7 角色系统 (character.js)

**CharacterSystem** — 3 个角色定义：

- `allCharacters[]` — 含基础属性、武器槽位、解锁条件、图标
- `applyToPlayer(player, id)` — 将角色属性覆写到玩家对象

### 3.8 解锁系统 (unlock.js)

**UnlockSystem** — localStorage 跨局持久化：

- 统计项：totalKills/totalLevels/maxLevel/totalMaterials
- 解锁集：unlockedWeapons (Set) / unlockedCharacters (Set)
- `endSession()` — 结算时更新统计 + 检查新解锁
- 武器解锁门槛：累计通关或累计击杀
- 角色解锁门槛：单局最高通关或累计击杀

### 3.9 UI系统 (ui.js)

**UISystem** — DOM 操作驱动的界面层：

- 角色选择列表渲染
- 商店商品网格渲染（武器/道具区分样式）
- HUD 实时更新（血条/经验条/关卡/金币/击杀）
- 升级选择弹窗
- 结算界面（统计 + 本局武器 + 新解锁动画）

### 3.10 渲染系统 (renderer.js)

**Renderer** — Canvas2D 渲染管线：

```
beginFrame(player)
  ├── 清屏 (#0a0a0f)
  ├── 计算摄像机偏移 (以玩家为中心)
  └── ctx.save + translate
      ├── drawBackground() → 网格 + 装饰点
      ├── drawWorldBounds() → 虚线边界
      ├── drawMaterial() → 金币浮动渲染
      ├── drawEnemy() → 6种形状 + 血条
      ├── drawParticle() → 圆/火花/辉光
      ├── drawPlayer() → 渐变圆 + 眼睛
      └── drawBullet() → 9种武器视觉区分
  └── ctx.restore
drawHUDEffects() → 低血量红色闪烁
```

DPI 适配：使用 `ctx.setTransform(dpr, 0, 0, dpr, 0, 0)` 避免缩放累积。

### 3.11 粒子系统 (particle.js)

**ParticleSystem** — 对象池模式：
- 类型：circle / spark / glow
- 预制特效：explosion / enemyDeath / pickup（3层粒子：火花+光晕+闪光）/ levelUp

### 3.12 音频系统 (audio.js)

**AudioSystem** — Web Audio API 音效 + 文件 BGM 播放：

```
初始化: GameEngine.startGame() → AudioSystem.init()
  ├── AudioContext 创建（需用户交互触发，如点击"开始游戏"）
  ├── 异步加载 sounds/tracks.json 动态填充 BGM_TRACKS
  ├── BGM 音量通道: _bgmGain (默认 0.06)
  └── SFX 音量通道: _sfxGain (默认 0.15)
```

**BGM 曲目清单：** 在 `sounds/tracks.json` 中定义，格式为 `[{ id, file, name }]`。
`audio.js` 在 `init()` 时自动 `fetch()` 加载此清单，动态填充 `BGM_TRACKS` 数组。
用户只需在 `tracks.json` 中添加条目即可增加曲目，无需修改代码。

播放优先级：音频文件（`fetch` + `decodeAudioData`）→ 程序化回退（赛博低音线）。
文件加载失败或 `file://` 协议限制时自动降级到程序化 BGM。

**音效类型 (play 方法)：**
| type | 波形 | 频率 | 时长 | 触发场景 |
|------|------|------|------|---------|
| `shoot` | square | 800→200Hz | 80ms | 射击/远程武器攻击 |
| `enemy_hit` | sine | 400→100Hz | 60ms | 子弹命中敌人 |
| `enemy_die` | noise(lowpass) | 800→100Hz | 150ms | 敌人死亡 |
| `hurt` | sawtooth | 150→50Hz | 150ms | 玩家受伤 |
| `pickup` | sine | 880→1320Hz | 150ms | 拾取医疗包/宝箱 |
| `coin` | sine | 1200→1800Hz | 100ms | 拾取金币 |
| `explosion` | noise+低频sine | 400→50Hz | 300ms | 爆炸范围伤害 |
| `levelup` | sine琶音 | C5→E5→G5→C6 | 4×200ms | 升级触发 |

**BGM 播放机制：**
- 启动时调用 `startBGM(trackId)` → 先尝试加载文件 BGM
- 使用 `fetch` + `decodeAudioData` 加载音频文件为 ArrayBuffer → `AudioBuffer`
- 通过 `BufferSourceNode(loop=true)` 循环播放，桥接到 `_bgmGain`
- 文件加载失败 → 自动回退到程序化赛博低音线 BGM
- `pauseBGM()` — 暂停 BGM（商店/菜单场景，保留曲目状态），清理节点但保留 `_bgmPlaying` 标记
- `resumeBGM()` — 恢复 BGM，从当前曲目头部重新开始
- `stopBGM()` — 完全停止并清理文件音频和程序化音频
- `switchBGM(trackId)` — 切换曲目（战斗/关卡切换用）

**程序化 BGM 回退：**
- 风格：赛博朋克风格循环低音线
- BPM：120，8 拍循环（~4秒/循环）
- 低音线：C2→D2→E2→D2（sawtooth + lowpass 200Hz）
- 节奏打击：每拍一个 noise(highpass 3000Hz) 短脉冲
- 循环机制：setTimeout 递归调度，重启游戏时自动清理旧循环

### 3.13 输入系统 (input.js)

**Input** — 键盘 + 鼠标：
- 键盘状态（持续按下 + 刚按下）
- 鼠标屏幕坐标 → 世界坐标转换

## 四、数据流

### 战斗帧循环（60fps）

```
Input.poll()
  → PlayerSystem.update(dt, enemies)
      → 键盘移动
      → 击退衰减
      → 自动攻击（所有武器）→ 播放 shoot / enemy_hit 音效
          → _fireWeapon() → BulletSystem.create()
      → 回血
      → 受击 → PlayerSystem.takeDamage() → 播放 hurt 音效
  → EnemySystem.update(dt, player)
      → AI 行为 → 碰撞伤害
      → 自爆 → _explodeDamage() → 播放 explosion 音效
  → BulletSystem.update(dt)
      → 跟踪/爆炸/过期
  → GameEngine._checkBulletCollisions()
      → 子弹 vs 敌人 → EnemySystem.takeDamage() → enemy_hit 音效
          → 死亡 → _handleEnemyKill() → dropMaterials() + addXP()
              → 播放 enemy_die 音效
      → 敌人子弹 vs 玩家 → PlayerSystem.takeDamage() → hurt 音效
  → GameEngine._updateMaterials(dt, player)
      → 金币吸引动画（距 < pickupRange 即飞向角色）
      → 到达 collectDist → 收集 → 播放 coin 音效 + 拾取粒子
  → ChestSystem.update(dt, player)
      → 宝箱吸引动画（距 < pickupRange 即飞向角色）
      → 到达 collectDist → 拾取 → 播放 pickup 音效
  → MedkitSystem.update(dt, player)
      → 医疗包吸引动画（距 < pickupRange 即飞向角色）
      → 到达 collectDist → 拾取 → 播放 pickup 音效
  → ParticleSystem.update(dt)
  → WaveSystem.update(dt, player)
      → 生成敌人
      → 关卡计时 → waveTransitioning → 残余宝箱/血包自动收集
  → 升级触发 → 播放 levelup 音效
  → Renderer 渲染帧
```

### 商店流

```
WaveSystem 30秒到
  → EnemySystem.clear() (掉落剩余材料)
  → ShopSystem.generateItems()
  → state = 'shopping'
  → UISystem.showShop()
  → 玩家点击购买 → ShopSystem.buyItem()
  → 玩家点击"进入下一关"
      → 如有升级待处理 → 升级界面
      → 否则 → WaveSystem.startNextLevel() → state = 'playing'
```

## 五、性能优化策略

1. **对象池** — bullet.js 和 particle.js 使用对象池，避免频繁 GC
2. **局部渲染** — Background 只计算可视区域的网格，不绘制全图
3. **索引逆序遍历** — 所有 bullet/particle/enemy 循环使用 `for (let i = arr.length-1; i >= 0; i--)`，支持安全的 splice 删除
4. **dt 限幅** — `Math.min(rawDt, 0.05)` 防止大帧跳跃（如切标签页回来）
5. **CSS backdrop-filter** — 菜单使用 blur(15px) 而非实时 Canvas 模糊，减少 GPU 开销

## 六、浏览器兼容性

- 使用 `requestAnimationFrame`、`Canvas2D`、`localStorage`
- 使用 `backdrop-filter`（需要 Chromium/WebKit 支持）
