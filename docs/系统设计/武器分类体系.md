# 武器分类体系 v1.0

> **源数据**: `Docs2/近战武器大全.md` (10 大类) + `Docs2/射击武器大全.md` (3 大类) + `Docs2/魔法武器大全.md` (9 大元素) + BroTato 8 类 (Blade/Blunt/Heavy/Precise/Primitive/Support/Elemental/Medical) + 现有项目 6 标签 (melee/gun/bow/magic/medic/lance)
>
> **定位**: 游戏可实施分类,**不是历史百科的搬运**。所有词汇只保留游戏中能"看出区别"的细粒度,合并掉同质化条目。
>
> **版本**: v1.0 · 2026-06-07
>
> **配合文档**: [`近战武器系统.md`](./近战武器系统.md) · [`射击武器系统.md`](./射击武器系统.md) · [`魔法武器系统.md`](./魔法武器系统.md) · [`角色-class偏好.md`](./角色-class偏好.md)

## 一、设计目标

1. 在现有 **6 标签 (tag)** 之上叠加 **1 级 class (12)** 和 **2 级 class_2 (30)** 维度
2. 角色偏好从"只挑 tag"升级为"挑 class_1 → class_2"两层,实现"剑客拿武士刀(满) vs 拿长剑(0.85) vs 拿斧(0.50) vs 拿枪(0.10)"4 元阶梯
3. **无破坏流程**: 旧字段全部保留,新字段叠加,旧逻辑不删
4. 数据驱动: 词汇表 + 映射表全部 JSON 化,不动 JS 业务代码

## 二、1 级 class (12 类)

| class | 中文 | icon | 包含的 2 级 | 备注 |
|-------|------|------|------------|------|
| **Blade** | 刀剑 | ⚔️ | longsword, katana, saber, scimitar, dagger, rapier, machete | 单刃直刃类,Brotato 沿用 |
| **Blunt** | 钝器 | 🔨 | mace, hammer, club, staff, flail | 锤/棍/锏/法杖(物理形态) |
| **Heavy** | 重型 | 🪨 | greataxe, warhammer, halberd, pike, lance, greatsword | 双手大型,Brotato 沿用 |
| **Precise** | 精确 | 🎯 | dagger, rapier, kris, stiletto | 匕首/细剑,高暴击流 |
| **Bow** | 弓 | 🏹 | longbow, recurve, composite | 从 Brotato Precise 拆出 |
| **Crossbow** | 弩 | 🎯 | handcrossbow, heavycrossbow, repeating | 从 Brotato 拆出独立 |
| **Gun** | 枪 | 🔫 | pistol, revolver, smg, rifle, sniper, shotgun, lmg, hmg | 从 Brotato Precise 拆出 |
| **Explosive** | 爆炸 | 💥 | grenade, rocket, mine, bomb | 从 Brotato Elemental 拆出 |
| **Elemental** | 元素 | 🔮 | fire, ice, lightning, wind, force, earth, shadow, holy, chaos | 9 元素,见 [魔法武器系统](./魔法武器系统.md) |
| **Medical** | 医疗 | 💊 | heal, regen, buff, shield | 治疗/护盾,见 [魔法武器系统](./魔法武器系统.md) |
| **Support** | 支援 | 🛡️ | shield, aegis, barrier | 防御向 |
| **Primitive** | 原始 | 🪓 | stone, bone, club | 简易武器,Brotato 沿用 |

> **与 BroTato 8 类的差异**: 多了 `Bow` / `Crossbow` / `Gun` / `Explosive` 4 个独立类,合并了 `Heavy + Support`。
>
> **设计原则**: 1 级 class 应当**让玩家一眼看出区别**。弓 vs 弩 vs 枪 不能合并成一个 `Ranged`,因为它们机制差异巨大(弓要蓄力、弩要装填、枪要换弹)。

## 四、tag → class 映射 (无破坏,核心)

> **目的**: 让现有 48 把武器自动获得 1 级 class,不需要逐个改 weapon JSON。

| weapon.tag | 默认 class (1级) | 备注 |
|------------|------------------|------|
| `melee` | `Blade` (单手) | 默认 `Blade`,**重型** (斧/锤/戟/枪) 显式标 `Heavy` |
| `gun` | `Gun` | 全部映射到 Gun |
| `bow` | `Bow` (标准) / `Crossbow` (机械) | 按 `pierce >= 2` 或 `damageMult >= 2.0` 区分 |
| `magic` | `Elemental` (法术) / `Explosive` (爆炸) | 按 `behavior === 'explode'` 区分 |
| `medic` | `Medical` | 全部映射到 Medical |
| `lance` | `Heavy` | 全部映射到 Heavy (lance class_2) |

> **特殊处理**:
> - `tag = melee` 的武器默认 `Blade`,**除非显式声明** `class = Heavy` / `Blunt` / `Precise`
> - **移除 `slots` 字段** (v1.1 简化): 所有武器占 1 槽,重型武器通过 `damageMult` / `price` / `attackSpeed` 平衡
> - **JSON 覆盖优先级**: weapon.class 显式值 > 上述默认映射规则

## 五、4 元阶梯伤害公式

```js
// 在 src/engine/formula.js 替换 _isTagMatched
// 新增 _matchWeaponClass(weapon, character) → number

function _matchWeaponClass(weapon, character) {
    const tags = character.tags || [];                       // 兼容旧 (7 大类)
    const pref1 = character.preferredClasses || [];           // 1 级偏好
    const pref2 = character.preferredClasses_2 || [];         // 2 级偏好

    // 阶梯 1: 2 级匹配 (满效果)
    if (pref2.includes(weapon.class_2)) return 1.0;

    // 阶梯 2: 1 级匹配 (次擅长)
    if (pref1.includes(weapon.class)) return 0.85;

    // 阶梯 3: tag 匹配 (同流派半效)
    if (tags.includes(weapon.tag)) return 0.50;

    // 阶梯 4: 都不匹配 (严重惩罚)
    return 0.10;
}
```

### 5.1 实测示例 (剑客 swordsman)

| 武器 | weapon.class | weapon.class_2 | weapon.tag | 阶梯 | 倍率 |
|------|-------------|---------------|------------|------|------|
| 武士刀 `katana` | Blade | katana | melee | ✓2级 | **1.00** |
| 长剑 `sword` | Blade | longsword | melee | ✓1级 | **0.85** |
| 能量斧 `axe` | Heavy | greataxe | melee | ✓tag | **0.50** |
| 基础手枪 `pistol` | Gun | pistol | gun | ✗ | **0.10** |

### 5.2 实测示例 (枪手 gunslinger)

| 武器 | 阶梯 | 倍率 |
|------|------|------|
| 狙击枪 `sniper` (Gun.sniper) | ✓2级 | 1.00 |
| 步枪 `rifle` (Gun.rifle) | ✓1级 | 0.85 |
| 霰弹枪 `shotgun` (Gun.shotgun) | ✓1级 | 0.85 |
| 长弓 `bow` (Bow.longbow) | ✓tag | 0.50 |
| 武士刀 `katana` | ✗ | 0.10 |

```

       ↓
scripts/csv2json.cjs (同步更新)
       ↓

       ↓
```

## 七、迁移计划 (5 步,无破坏)

### 7.1 字段变更总览 (v1.1)

| 文件 | 旧字段 | 新字段 | 备注 |
|------|--------|--------|------|
| `csv/weapons.csv` | `slots: 1-3` | **移除** | 统一 1 槽/武器 |
| `csv/weapons.csv` | (无) | `class` (新增) | 1 级 class 显式标注 |
| `csv/weapons.csv` | (无) | `class_2` (新增) | 2 级 class_2 显式标注 |
| `csv/characters.csv` | `weaponSlots: 4-6` | `maxWeapons: 4-6` | 字段重命名,值不变 |
| `csv/characters.csv` | (无) | `preferredClasses` (新增) | 1 级偏好数组 |
| `csv/characters.csv` | (无) | `preferredClasses_2` (新增) | 2 级偏好数组 |
| `csv/characters.csv` | (无) | `crossbowman` 角色 (新增) | v1.1 新增弩手 |

### 7.2 5 步执行顺序

1. **CSV 同步**:
   - `csv/weapons.csv` 加 `class` / `class_2` 列;**删除** `slots` 列
   - `csv/characters.csv` 加 `preferredClasses` / `preferredClasses_2` / `crossbowman` 行;`weaponSlots` 改 `maxWeapons`

2. **JSON 同步**: 重跑 `scripts/csv2json.cjs` 生成 `src/data/weapons.json` + `src/data/characters.json`

3. **Bundle 同步**: 更新 `src/data/data-bundle.js` 内嵌 JSON,**改 `?v=` 时间戳**

4. **代码接入**:
   - `tags.js` 加 `CLASS_DEFS` / `CLASS_2_DEFS` / `_matchWeaponClass`
   - `character.js` 加偏好字段
   - **删除** `slots` 相关代码 (无破坏: 旧武器没 slots 字段时默认 1)

5. **公式升级**:
   - `formula.js` 加 `_matchWeaponClass` (4 元阶梯)
   - **保留** `_isTagMatched` (兜底)
   - ShopSystem / 装备系统统一改用 `maxWeapons` 而非 `weaponSlots`

> 每步都需要 `node --check` + `npm test` (基线 417/15) 守恒
