# 宝箱/升级抽卡金币刷新设计

> 创建: 2026-06-08 | 状态: 设计阶段, 待实施

## 目标

Brotato 风格: 宝箱抽卡和升级抽卡都可以用金币刷新选项, 让玩家有更多选择空间.

## 机制

### 刷新规则

| 场景 | 刷新费用 | 限制 |
|---|---|---|
| 宝箱抽卡 | 1 金币 | 每宝箱只能刷新 1 次 |
| 升级抽卡 | 1 金币 | 每级只能刷新 1 次 |

### 刷新逻辑

1. 玩家在抽卡界面看到 "刷新" 按钮
2. 点击 → 扣除 1 金币 → 重新生成选项
3. 新选项覆盖旧选项, 玩家不可撤销
4. 刷新后按钮变灰 (disable), 标记 "已刷新"

## 宝箱刷新

修改 `LootSystem.generateRewards`:
- 新增 `chestRefreshCount` 字段
- `pickupChest` 时记录 `refreshAvailable = true`
- 刷新按钮调用 `LootSystem.refreshChest(player)`

```js
// loot.js
refreshChest(player) {
    if (!this._currentChestType) return;
    if (player.materials < 1) return;  // 金币不足
    if (this._chestRefreshed) return;  // 已刷新过
    
    player.materials -= 1;
    this._chestRefreshed = true;
    this.currentRewards = this.generateRewards(this._currentChestType, player);
}
```

## 升级抽卡刷新

修改 `LevelUpSystem`:
- 新增 `refreshAvailable` 字段
- 刷新按钮调用 `LevelUpSystem.refreshCards(player)`

```js
// levelup.js
refreshCards(player) {
    if (player.materials < 1) return;
    if (this._cardsRefreshed) return;
    
    player.materials -= 1;
    this._cardsRefreshed = true;
    this.currentCards = this.generateCards(player);
    // 刷新 UI
    UISystem.showLevelUpCards(this.currentCards);
}
```

## UI 改动

抽卡界面 (宝箱/升级) 在选项区域下方增加:
```
[🔄 刷新 (1 金币)]  ← 已刷新后变灰显示 "🔄 已刷新"
```

| `src/engine/levelup.js` | +refreshCards, +_cardsRefreshed |
| `src/cyberblade/ui.js` | 抽卡界面 + 刷新按钮 |

## 注意事项

- 刷新费用固定 1 金币, 不随难度变化
- 每次打开宝箱/升级, 刷新状态重置
- 金币不足时按钮显示为红色禁用态
- 刷新后的新选项仍然受流派偏向影响
