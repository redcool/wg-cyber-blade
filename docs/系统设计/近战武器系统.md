## 二、近战武器核心机制 (5 维度)

### 2.1 攻击方式 (behavior)

| behavior | 中文 | 适用 class | 现有武器 |
|----------|------|-----------|----------|
| `melee_thrust` | 突刺 | Blade.dagger/rapier, Heavy.pike/lance | sword, katana, spear, pike, lance, trident |
| `melee_sweep` | 横扫 | Blade.longsword, Blunt, Heavy.greataxe | plasma, axe, hammer, whip |
| `melee_combo` | 多连击 | Blade.dagger, Precise | dagger (2连), claws (3连) |
| `melee_throw` | 投掷 | Precise.javelin | (新增) |

> **设计要点**:
> - thrust 武器 (刺剑/枪) → 直线穿透,适合对付密集敌群
> - sweep 武器 (斧/锤) → 范围扇形,适合对付单体 Boss
> - combo 武器 (匕首) → 高 DPS,但低单次伤害,适合 build 成型后

### 2.2 攻击范围 (meleeRange)

| 范围段 | px | 适用 class | 设计意图 |
|--------|-----|-----------|---------|
| 短 | 40~60 | dagger, claws, rapier | 贴身,高攻速,需走位 |
| 中 | 80~100 | sword, katana, axe | 平衡 |
| 长 | 120~150 | spear, whip, halberd | 安全输出 |
| 超长 | 180~250 | pike, lance, trident | 骑枪流,几乎不接触 |

### 2.3 穿透 (pierce)

| 等级 | 值 | 适用 class | 备注 |
|------|-----|-----------|------|
| 无 | 0 | axe, hammer, claws | 纯单体 |
| 弱 | 1~2 | sword, katana | 偶尔穿透 |
| 中 | 3~4 | spear, halberd | 稳定穿透 |
| 强 | 5~6 | pike, lance, trident | 骑枪专属,清场 |

### 2.4 击退 (knockback)

| 来源 | class | 设计意图 |
|------|-------|---------|
| 0 (无) | dagger, rapier | 高 DPS 不打断敌群 |
| 200~300 (轻) | sword, katana | 创造空间 |
| 400~500 (中) | axe, halberd | 安全输出 |
| 600+ (强) | hammer, warhammer | 击退 + 高伤,代价是慢速 |

### 2.5 特殊效果 (与元素联动)

| 效果 | 适用 class_2 | 现有武器 |
|------|-------------|----------|
| **灼烧** (burnDps) | chainsaw (刃+热), fire_blade | chainsaw |
| **流血** (bleedDot, 暂未实现) | dagger, kris | (新增) |
| **缠绕** (slow + root) | whip, chain | whip (待扩展) |
| **暴击倍增** (critDmg ×3) | katana | katana |
| **击晕** (stun) | mace, hammer | (新增) |
| **格挡** (block) | shield | (新增) |

## 三、近战 class 与 1 级 tag 的关系

| 1 级 class | 关联 weapon.tag | tagSynergy 方向 |
|------------|----------------|----------------|
| `Blade` | melee | 吸血 + 暴伤 |
| `Blunt` | melee | 击退 + 护甲 (设计意图) |
| `Heavy` | melee, lance | 穿透 + 射程 (骑枪) |
| `Precise` | melee, lance | 暴击 + 闪避 (高机动) |
| `Support` | (无独立 tag) | 防御向, 走 Support 系 synergy (待设计) |
| `Primitive` | melee | (原始系,简化属性) |

> **关键**: `tag = melee` 在 synergy 系统中是 1 个大组,但通过 `class` 可以让"剑客 vs 钝器使用者"在同一 melee 大组内有差异化的 build 路线。

## 五、设计原则 (KISS)

1. **不要照搬百科**: 50+ 子类合并为 30 个 class_2,因游戏机制上无法区分
2. **优先行为,其次外观**: 两把剑如果攻击方式/范围/穿透/击退都一样,共用 class_2
3. **1 级 class 决定 Build 路线, 2 级 class 决定手感**: 剑客 (Blade 偏好) 拿所有 Blade 类武器都是 0.85,拿具体哪一把 (longsword/katana) 是手感差异
4. **新增武器流程**: 选 1 级 class → 选 2 级 class_2 → 写 5 维度参数 (behavior/range/pierce/knockback/特殊)
5. **v1.1 槽位简化**: 所有武器占 1 槽 (移除 `slots` 字段),重武器通过 damageMult / price / knockback 平衡
