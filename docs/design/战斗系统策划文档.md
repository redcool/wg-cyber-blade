# 战斗系统策划文档 v1.2

> 本文档定义 CYBER BLADE（赛博土豆）的全部战斗机制，供开发和调优参考。

---

## 1. 核心战斗循环

```
每帧循环:
  1. 玩家移动 (WASD + facingAngle 更新)
  2. 每个武器独立检查:
     a. 冷却就绪? (cooldownTimer <= 0)
     b. 范围内有敌人? (近战用 meleeRange, 远程用 attackRange)
     c. ✅ → 前方向对准目标 → 开火
  3. 子弹飞行 + 碰撞检测
  4. 敌人 AI 更新
  5. 拾取/材料/宝箱检测
```

### 1.1 方向系统

| 概念 | 值 | 说明 |
|------|-----|------|
| **forward** | 0 rad（正右方） | 所有武器图标的默认朝向。右方 = 正方向。 |
| **facingAngle** | `atan2(dy, dx)` | 玩家朝向。由 WASD 移动方向决定。 |
| **attackTargetAngle** | `atan2(target.y - p.y, target.x - p.x)` | 开火时瞄准目标的方向。 |
| **orbitalAngle** | `(i/count) * 2π - π/2` | 第 i 个武器在 360° 轨道上的角度。 |

**行为规则：**
- 移动时：`facingAngle` 跟随 WASD 方向
- 静止攻击时：`facingAngle` 转向攻击目标方向（使角色面朝敌人）
- 武器图标默认朝外（沿 orbitalAngle 方向）
- 攻击时：武器图标朝目标方向旋转

### 1.2 攻击流程

```
[武器冷却就绪]
    ↓
[查找范围内最近敌人] ← 每个武器独立查找
    ↓
[设置 facingAngle = 目标方向]
[设置 attackTargetAngle = 目标方向]
    ↓
[根据武器 behavior 调用对应开火函数]
    ↓
[设置 cooldownTimer = cd]
    ↓
[子弹/近战伤害处理]
```

## 2. 武器系统

### 2.1 武器属性

```
{
  id: string,          // 唯一标识
  name: string,        // 显示名称  tag: string,          // 标签 (melee/gun/bow/magic/medic/lance)
  behavior: string,    // 行为模式 (见 2.3)
  slots: number,       // 占位格数 (1~3)
  damageMult: number,  // 伤害倍率
  attackSpeedMult: number, // 攻速倍率
  bulletSpeed: number, // 弹速
  bulletCount: number, // 弹数
  pierce: number,      // 穿透次数
  spread: number,      // 散射弧度
  meleeRange: number,  // 近战范围
  attackRange: number, // 远程射程
  // 特殊属性...
}
```

### 2.2 等级与品质

```
等级加成: damageMult *= 1 + (level - 1) * 0.25
品质加成: T1=1.0, T2=1.3, T3=1.6, T4=2.0

最终伤害 = 玩家基础伤害 × damageMult × 品质加成 × 等级加成
```

| 品质 | 名称 | 颜色 | 倍率 | 最低波次 | 出现权重 |
|------|------|------|:----:|:--------:|:--------:|
| T1 | 普通 | `#aaaaaa` | ×1.0 | 1 | 45 |
| T2 | 优秀 | `#4488ff` | ×1.3 | 3 | 30 |
| T3 | 稀有 | `#aa44ff` | ×1.6 | 6 | 18 |
| T4 | 传说 | `#ff6600` | ×2.0 | 10 | 7 |

### 2.3 武器行为模式

| behavior | 说明 |
|----------|------|
| `bullet` | 标准子弹，直线飞行 |
| `spread` | 散射，多弹体扇形分布 |
| `laser` | 快速直线，3发微扩散 |
| `shock` | 电击，穿透+连锁弹跳 |
| `melee_sweep` | **近战横扫** — 180° 扇形范围，直接命中检测 |
| `melee_thrust` | **近战突刺** — 直线穿透，窄扇形命中检测（骑枪武器固定使用此行为） |
| `explode` | 爆炸弹，命中后范围伤害 |
| `frost` | 冰霜，减速+冰爆 |
| `homing` | 跟踪弹，自动追踪最近敌人 |
| `spray` | 喷射，锥形多弹体（火焰/毒/冷气） |
| `heal_bullet` | 治愈弹，命中回血 |
| `shield_aura` | 圣光盾，光环治疗 |

### 2.4 近战武器：动态选择（横扫/突刺）

```
targetDist < meleeRange * 0.45  →  melee_sweep（近距离横扫）
targetDist >= meleeRange * 0.45 →  melee_thrust（中远距离突刺）
```

#### 横扫 (melee_sweep)
```
范围: 180° 扇形
伤害: 范围全额伤害
穿透: 范围内容所有敌人
冷却: 正常武器冷却
特效: 弧形轨迹火花 + 武器沿弧线移动
```

#### 突刺 (melee_thrust)
```
范围: 直线 ~30px 宽
穿透: pierce 次（默认3次）
距离: 武器的 meleeRange
冷却: 正常武器冷却
特效: 武器沿直线飞出 + 命中火花
命中: 直接伤害，不生成子弹对象
```

### 2.5 武器冷却系统

```
每个武器拥有独立冷却:

cooldownTimer = (1.0 / 玩家攻速) × max(0.2, 武器攻速倍率)

- 多个武器独立循环检查冷却
- 每个武器各自检查自身射程内的敌人
- 冷却未到、或范围内无敌人 → 跳过此武器
- 不共享全局攻击计时器
```

### 2.6 近战武器攻速梯度 (v1.1 平衡)

近战武器按 `attackSpeedMult` 分为 5 个速度梯度，极差约 **7.2 倍**（最快 0.26s/击 → 最慢 1.87s/击）：

| 梯度 | attackSpeedMult | 范围 | 武器 | 有效冷却 | 攻击/秒 | 特征 |
|------|---------------|------|------|---------|--------|------|
| ⚡极速 | 0.15~0.35 | < 50px | 利爪 (0.30) | 0.26s | 3.8 | 三连爪击，极短距离高频攻击 |
| ⚡快速 | 0.36~0.50 | < 70px | 双持匕首 (0.38), 等离子刀 (0.50) | 0.35~0.50s | 2.0~2.9 | 持续压制型 |
| 🔄中等 | 0.51~0.70 | 60~85px | 链锯剑 (0.55), 能量剑 (0.60), 武士刀 (0.70) | 0.55~0.70s | 1.4~1.8 | 综合均衡型 |
| 🔨慢速 | 0.71~1.00 | 90~120px | 能量矛 (0.85), 能量斧 (1.00) | 0.94~1.18s | 0.85~1.1 | 厚重范围型 |
| 🐢极慢 | 1.01~1.40 | 90~150px | 能量鞭 (1.25), 重锤 (1.40) | 1.25~1.87s | 0.54~0.80 | 超慢致命一击型 |

> **冷却计算公式**（以初始玩家攻速 1.0 为基准）：
> `冷却 = (1.0 / 玩家攻速) × attackSpeedMult`
> 其中 `玩家攻速` 受武器 `mods.attackSpeedMult` 修正（如重锤 -25% 攻速）

### 2.7 武器轨道分布

```
360° 均匀环绕，从正上方(-π/2)开始:

angle = (i / count) × 2π - π/2

轨道距离:
iconSize = max(18, round(玩家半径 × 1.0 + slots × 5))
dist = 玩家半径 + 6 + iconSize × 0.55

最大 6 个武器。
```

## 3. 伤害系统

### 3.1 伤害计算

```
基础伤害 = 玩家 damage × 武器 damageMult × 品质加成 × 等级加成

暴击判定: random < critChance
暴击伤害: 基础伤害 × critMultiplier

最终伤害 = StatsSystem.calcDamageReduction(伤害, 护甲)
```

### 3.2 伤害类型

| 类型 | 效果 |
|------|------|
| **物理** | 直接扣血 |
| **燃烧 (burn)** | DOT，每秒 dps 伤害，最多 burnMaxStacks 层 |
| **冰冻 (slow)** | 减速，slowFactor 持续 slowDuration 秒 |
| **爆炸 (splash)** | 范围伤害，边缘衰减50% |
| **连锁 (chain)** | 弹跳至附近敌人，每跳伤害衰减20% |
| **跟踪 (homing)** | 自动修正弹道追踪敌人 |

### 3.3 击杀处理链

```
敌人 hp <= 0
    ↓
1. kills++（统计）
2. addXP(e.xpValue) → 可能触发升级
3. lifeSteal 吸血处理
4. _dropMaterials(e) → 掉落金币（10%概率2枚，90%概率1枚）
5. 精英 → ChestSystem.spawnChest(x, y, tier=1)
   Boss  → ChestSystem.spawnChest(x, y, tier=2)
6. ParticleSystem.enemyDeath() → 死亡特效
7. AudioSystem.play('enemy_die') → 死亡音效
```

### 3.4 燃烧系统

```
- 每层独立计时
- 最大叠加层数: 3~5（武器定义）
- 每秒伤害 = 层数 × dps
- 传播: _spreadBurn() 扩散至附近敌人（有扩散器时）
```

## 5. 敌人生成规则

### 5.1 敌人类型解锁

| 关卡 | 解锁 |
|------|------|
| 1 | basic |
| 2 | fast |
| 4 | tank |
| 6 | ranged |
| 10 | elite |

### 5.2 同时存在限制

| 类型 | 限制 |
|------|------|
| elite | 最多 1 个同时在场 |
| boss | 最多 1 个同时在场 |
| 其他 | 无上限（受 maxEnemies 限制） |

### 5.3 难度缩放

```
difficultyScale = 1 + (currentLevel - 1) × 0.15
spawnRate = max(0.4, 1.5 - (currentLevel - 1) × 0.04)
enemyCount = 4 + floor(currentLevel × 1.5)
```

### 5.4 Boss 生成

```
每 5 关（5/10/15/20...）:
  关卡第 4 秒时，在远处生成一个精英（5~10关）
  或 Boss（15关以上）
```

## 6. 渲染规范

### 6.1 武器图标渲染

```
- 默认方向: 右 = 0 rad（图标 PNG 朝右绘制）
- 默认朝向: 沿 orbitalAngle 朝外
- 攻击横扫: 武器沿 180° 弧线从目标左侧扫到右侧
- 攻击突刺: 武器沿攻击方向飞出约 6× 角色半径，再归位
```

### 6.2 角色朝向

```
- WASD 移动: facingAngle = atan2(dy, dx)
- 攻击时（无论是否移动）: facingAngle 覆盖为攻击目标方向
- 未移动且未攻击: 保持上一次的 facingAngle 值
- 角色 PNG 默认朝右，向左时水平翻转 (scaleX = -1)

优先级: WASD 移动设置 → 攻击覆盖（攻击优先级更高）
```

## 8. 数据流

```
每帧:
  PlayerSystem.update(dt)
    ↓ 查找敌人 → 检查冷却 → 开火 → 播放射击/近战音效
    ↓ 受击 → takeDamage() → 播放 hurt 音效
    ↓
  BulletSystem.update(dt) — 子弹飞行 + 跟踪
    ↓
  GameEngine._checkBulletCollisions()
    ↓ 命中 → enemy_hit 音效
    ↓ 击杀 → _handleEnemyKill() → 掉落金币/宝箱 → enemy_die 音效
    ↓
  EnemySystem.update(dt) — AI + 燃烧DOT
    ↓ 自爆 → _explodeDamage() → explosion 音效
    ↓
  GameEngine._updateMaterials(dt, player)
    ↓ 金币 < pickupRange → 飞向角色 → 收集 → coin 音效 + 拾取粒子
    ↓
  ChestSystem.update(dt, player)
    ↓ 宝箱 < pickupRange → 飞向角色 → 拾取 → pickup 音效
    ↓
  MedkitSystem.update(dt, player)
    ↓ 医疗包 < pickupRange → 飞向角色 → 拾取 → pickup 音效
    ↓
  WaveSystem.update(dt) — 生成敌人 + 关卡计时
    ↓ 关卡结束 → 残余宝箱/血包自动收集
    ↓
  升级 → 播放 levelup 音效
    ↓
  UISystem.updateHUD() — 刷新显示
```

## 9. 拾取吸引系统

### 9.1 概述

所有掉落物品（金币、医疗包、宝箱）在进入玩家 `pickupRange` 后会以恒速飞向角色，到达 `collectDist` 时自动收集。无需玩家主动按键拾取。

### 9.2 吸引参数

| 物品 | 吸引速度 | 收集距离 | 生命周期 | 实现位置 |
|------|---------|---------|---------|---------|
| 金币 (materials) | 300 px/s | 10 px | 30秒 | `main.js` _updateMaterials() |
| 医疗包 (medkit pickup) | 250 px/s | 10 px | 8秒 | `wave.js` MedkitSystem.update() |
| 宝箱 (chest) | 180 px/s | 12 px | 永久（直到拾取） | `wave.js` ChestSystem.update() |

### 9.3 吸引逻辑

```
每帧:
  for each item:
    dist = distance(item, player)
    if dist < collectDist:
      → 收集（金币加值/回血/宝箱入队）
      → 播放对应音效
      → 触发拾取粒子特效
    elif dist < pickupRange:
      → factor = dt × attractSpeed / max(1, dist)
      → item.x -= dx × factor
      → item.y -= dy × factor
```

### 9.4 关卡结束自动收集

当波次结束时（`waveTransitioning = true`），场景上所有未拾取物品强制收集：

| 物品 | 处理方式 |
|------|---------|
| 地面金币 | 全部消失（不清除已积累的金币数值） |
| 宝箱 | 自动拾取 → 加入 pendingChests 队列 → 后续展示奖励弹窗 |
| 医疗包 | 自动回血（直接加给玩家，不经过吸引动画） |

## 10. 音频系统

### 10.1 架构

音效使用 Web Audio API 程序化生成，BGM 优先加载 `sounds/` 目录中的音频文件，失败时回退到程序化生成。

- `AudioContext` 在用户点击「开始游戏」时创建（满足浏览器用户交互要求）
- 双通道：`_bgmGain`（背景音乐，默认音量 0.06）和 `_sfxGain`（音效，默认音量 0.15）
- BGM 曲目清单在 `sounds/tracks.json` 中定义，`audio.js` 在 `init()` 时自动 `fetch()` 动态加载
- BGM 文件通过 `fetch` + `decodeAudioData` 加载为 `AudioBuffer`，使用 `BufferSourceNode(loop=true)` 循环播放
- 进入商店/升级/宝箱奖励时自动 `pauseBGM()`，返回战斗时 `resumeBGM()`
- 各音效内部还有单独的 gain 倍率（0.4~1.0），用于调节相对响度

### 10.2 音效触发点

| 事件 | 音效类型 | 触发位置 |
|------|---------|---------|
| 远程武器攻击 | `shoot` | `player.js` _fireWeapon() |
| 近战武器攻击 | `enemy_hit` | `player.js` _fireWeapon() |
| 子弹命中敌人 | `enemy_hit` | `main.js` _checkBulletCollisions() |
| 敌人死亡 | `enemy_die` | `main.js` _handleEnemyKill() |
| 自爆者爆炸 | `explosion` | `enemy.js` _explodeDamage() |
| 玩家受伤 | `hurt` | `player.js` takeDamage() |
| 拾取医疗包/宝箱 | `pickup` | `wave.js` MedkitSystem / ChestSystem |
| 拾取金币 | `coin` | `main.js` _updateMaterials() |
| 升级 | `levelup` | `player.js` addXP() |

### 10.3 背景音乐

#### 曲目列表

BGM 曲目列表从 `sounds/tracks.json` 动态加载，`audio.js` 的 `init()` 时自动异步 `fetch()` 解析：

| 曲目 ID | 文件 | 名称 | 用途 |
|---------|------|------|------|
| `combat1` | `sounds/bgm_1.m4a` | 战斗主题1 | 普通战斗循环（首个曲目） |
| `combat2` | `sounds/bgm_2.m4a` | 战斗主题2 | 普通战斗循环 |

添加新 BGM 只需在 `sounds/tracks.json` 中追加条目：

```json
{ "id": "boss", "file": "sounds/bgm_boss.m4a", "name": "Boss 战" }
```

> `tracks.json` 由 `audio.js` 在 `init()` 时异步加载，无需修改代码。

#### 播放机制

```
startBGM(trackId)
  └─ 从 BGM_TRACKS（动态加载自 tracks.json）查找曲目
      └─ fetch(track.file) → arrayBuffer → decodeAudioData
          ├─ 成功 → BufferSourceNode(loop=true) → connect(_bgmGain) → 循环播放
          └─ 失败 → _fallbackToProgrammatic()
                    └─ 赛博低音线循环（setTimeout 递归）

pauseBGM()
  └─ 停止 BGM 节点，保留 _bgmPlaying=true 标记

resumeBGM()
  └─ 从当前曲目头部重新 startBGM(_currentTrackId)
```

1. **文件 BGM（首选）**：使用 `fetch` + `decodeAudioData` 加载
   - 支持 `file://` 和 `http://` 协议（纯 Web Audio API，绕过 `createMediaElementSource` 的 CORS 限制）
   - 通过 `BufferSourceNode` 的 `loop=true` 实现循环播放
   - 音量跟随 `_bgmGain` 全局 BGM 音量
2. **程序化 BGM（回退）**：文件加载失败或曲目列表为空时自动降级
   - 风格：赛博朋克低音线 + 节奏打击
   - BPM：120（~4秒/8拍循环）
   - 低音线：C2→D2→E2→D2 的 sawtooth 波，经 lowpass 200Hz 滤波
   - 打击：每拍一个 noise 短脉冲，经 highpass 3000Hz 滤波
   - 递归调度：setTimeout 每次循环结束后重新调度下一轮

#### 商店 BGM 控制

| 事件 | 操作 | 触发位置 |
|------|------|---------|
| 关卡结束 → 进入商店 | `AudioSystem.pauseBGM()` | `main.js` `_updatePlaying()` |
| 关卡结束 → 奖励选择 | `AudioSystem.pauseBGM()` | `main.js` `_updatePlaying()` |
| 关卡结束 → 升级界面 | `AudioSystem.pauseBGM()` | `main.js` `_updatePlaying()` |
| 关闭商店 → 进入下一关 | `AudioSystem.resumeBGM()` | `main.js` `closeShop()` |

#### 曲目切换

`switchBGM(trackId)` 用于多曲目场景：
- 停止当前 BGM，清理音频资源
- 加载并播放新曲目
- 如果当前曲目已停止，调用 `switchBGM()` 的行为与 `startBGM()` 一致

#### 重启保护

`startGame()` 先调用 `stopBGM()` 清理旧循环/文件引用，再重新开始，防止死后重启残留。

### 10.4 可调参数

| 参数 | 位置 | 默认值 | 说明 |
|------|------|--------|------|
| BGM 音量 | `audio.js` _bgmGain.gain.value | 0.06 | 0~1，同时影响文件和程序化 BGM |
| SFX 音量 | `audio.js` _sfxGain.gain.value | 0.15 | 0~1 |
| BGM 曲目清单 | `sounds/tracks.json` | `[{ id, file, name }]` | 曲目定义列表（动态加载） |
| BGM 文件路径 | `tracks.json` 中的 file 字段 | `sounds/bgm_1.m4a` | 曲目文件路径 |
| 程序化 BPM | `_playBGMLoop()` bpm | 120 | 节拍速度（回退模式） |
| 程序化低音线 | `bassNotes` | [65.41, 73.42, 82.41, 73.42] | C2-D2-E2-D2（回退模式） |
