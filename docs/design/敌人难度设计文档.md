## 一、核心难度哲学

- **逐关递增**：每一关都比前一关更难，但给玩家充分适应的空间
- **三轴增长**：**数量↑** + **数值↑** + **种类↑** 三轴同步上升
- **可读性**：玩家能清晰感知到"更难了"，但不会觉得"不公平"
- **无限持续**：支持无限关卡循环，难度持续增长

## 二、波次控制（每秒刷怪数）

### 2.1 核心概念

不再使用固定间隔×单次1怪的简单模式，而是：

```
每次生成操作（Spawn Tick）产出多个怪：
  spawnsPerTick = 根据关卡计算的单次生成数量
```

| 关卡 | `spawnsPerTick` | 举例 |
|------|----------------|------|
| 1 | 1 | 每次生成 1 个 |
| 2 | 1~2 (随机) | 每次生成 1 或 2 个 |
| 3 | 1~2 | 每次生成 1 或 2 个 |
| 4 | 2~3 | 每次生成 2 或 3 个 |
| 5~6 | 2~3 | — |
| 7~9 | 3~4 | — |
| 10~14 | 3~5 | — |
| 15~19 | 4~6 | — |
| 20+ | 5~8 | 高难度大量刷怪 |

### 2.2 计算方式

```javascript
function getSpawnsPerTick(level) {
    if (level === 1) return 1;
    if (level === 2) return 1 + Math.random() < 0.5 ? 1 : 0; // 50%几率出2个
    const base = Math.min(level, 20);
    const min = Math.max(1, Math.floor(base * 0.25));
    const max = Math.max(1, Math.floor(base * 0.35));
    return min + Math.floor(Math.random() * (max - min + 1));
}
```

简化公式：
```javascript
spawnsPerTick = Math.max(1, Math.floor(level * 0.25) + (Math.random() < 0.3 ? 1 : 0));
```

## 三、敌人数值缩放

### 3.1 敌人等级概念

每个敌人实例有一个 `level` 属性，由当前关卡决定：

```javascript
enemy.level = waveLevel;  // 与关卡一致
```

### 3.2 基础属性缩放公式

```
HP:        baseHP × (1 + (level - 1) × 0.12)
Damage:    baseDMG × (1 + (level - 1) × 0.10)
Speed:     baseSpeed × (1 + (level - 1) × 0.04)
XP:        floor(baseXP × (1 + (level - 1) × 0.10))
Materials: floor(baseMat × (1 + (level - 1) × 0.08))
```

### 5.2 类型生成权重（随关卡调整）

早期关卡主要生成基础类型，后期增加高难度类型的比例：

```javascript
// 类型生成权重表
const typeWeights = {
    basic:   { early: 40, mid: 25, late: 15 },  // 早期多，后期减少
    fast:    { early: 25, mid: 20, late: 15 },
    tank:    { early: 0,  mid: 15, late: 15 },
    ranged:  { early: 0,  mid: 12, late: 12 },
    exploder:{ early: 15, mid: 10, late: 8 },
    healer:  { early: 0,  mid: 8,  late: 8 },
    mortar:  { early: 0,  mid: 5,  late: 7 },
    blinker: { early: 0,  mid: 5,  late: 7 },
    elite:   { early: 0,  mid: 0,  late: 3 },   // 随特殊生成
    boss:    { early: 0,  mid: 0,  late: 0 },   // 随特殊生成
};

// early = L1~5, mid = L6~14, late = L15+
function getTypeWeight(type, level) {
    const phase = level <= 5 ? 'early' : (level <= 14 ? 'mid' : 'late');
    return typeWeights[type][phase];
}
```

简化版：随机从 `_availableTypes` 中选取，但增加特定类型的出现概率控制：

```javascript
// 百分比权重（基于当前可用类型池）
const weights = {};
for (const t of types) {
    switch(t) {
        case 'basic':    weights[t] = 30; break;
        case 'fast':     weights[t] = 20; break;
        case 'exploder': weights[t] = level <= 10 ? 18 : 10; break;
        case 'tank':     weights[t] = level <= 10 ? 10 : 15; break;
        case 'healer':   weights[t] = level <= 8 ? 0 : 12; break;
        case 'ranged':   weights[t] = level <= 8 ? 0 : 12; break;
        case 'mortar':   weights[t] = level <= 10 ? 0 : 8; break;
        case 'blinker':  weights[t] = level <= 10 ? 0 : 8; break;
        case 'elite':    weights[t] = 0; break;  // 特殊生成
        case 'boss':     weights[t] = 0; break;  // 特殊生成
    }
}
```

## 六、完整生成流程（伪代码）

```javascript
// 每帧调用
function update(dt) {
    waveTimer += dt;
    spawnTimer += dt;

    const interval = getSpawnInterval(waveLevel);
    if (spawnTimer >= interval) {
        spawnTimer = 0;
        const count = getSpawnsPerTick(waveLevel);
        for (let i = 0; i < count; i++) {
            if (aliveCount >= maxSimultaneous) break;
            if (shouldSpawnEliteOrBoss()) continue;
            const type = pickWeightedType();
            const pos = getSpawnPosition(player);
            const enemy = EnemySystem.create(type, pos.x, pos.y, waveLevel);
        }
    }

    // 精英/Boss特殊生成（每5关）
    if (isEliteBossWave() && !bossSpawned && waveTimer > 4) {
        spawnEliteOrBoss();
    }
}
```
