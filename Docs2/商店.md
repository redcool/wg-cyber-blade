这几块属于 Roguelike 核心循环的奖励系统，也是驱动玩家“上瘾感”的关键。
我们可以把它们整合到你已经有的 属性 / 武器 /道具 /流派系统 中，让每一局的成长和选择都更丰富。

我给你做一个完整框架，直接可用在原型里：

一、商店系统（Shop System）
1️⃣ 核心理念
玩家在每波/每关结束后获得金币/材料
可选择购买武器、道具或角色升级
每件商品都和当前流派/属性有联动
2️⃣ 商品分类
分类	内容	设计注意
武器	当前可用武器 + 新武器	稀有度影响价格
道具	属性型/机制型道具	根据流派偏向出现
临时增益	一局内生效的buff	不破坏系统平衡
材料/资源	用于升级或强化	推动滚雪球或策略选择
3️⃣ 商店刷新机制
每次进商店，随机刷新 3~5 件商品
稀有度影响刷新概率：
Common 60%
Rare 25%
Epic 10%
Legendary 5%
可设计“折扣/幸运值”机制，增加策略感
4️⃣ 商店货币体系
金币（常规购买）
关键材料（升级、抽卡）
“幸运币”（可影响稀有刷新，或保底机制）
二、角色升级免费抽卡（Level-Up / Perk System）
1️⃣ 核心理念
每次角色升级时，玩家从随机选项中挑选一项加成
每个加成和流派、武器、道具都可以联动
保持“每次升级都很爽”的感受
2️⃣ 抽卡机制
每次升级随机提供 3~5 张卡
卡片类型：
属性加成（+HP、+Damage、+Speed 等）
武器/道具加成（+1武器插槽、特定道具增强）
特殊机制（OnHit / OnKill 触发强化）
每张卡可有随机化的数值（如+5%~+10% Damage）
3️⃣ 卡片成长与稀有度
稀有度	概率	数值范围
Common	60%	+5~10%
Rare	25%	+10~20%
Epic	10%	+20~30%
Legendary	5%	+30~50% + 特殊机制
4️⃣ 战略设计
每次升级可形成“流派强化”，例如：
你已经是暴击流，升级出现的卡片偏向暴击加成
你是工程流，升级偏向 Tech 属性或召唤能力
三、宝箱系统（Loot Chest System）
1️⃣ 核心理念
宝箱是随机奖励，但可设计保底+策略选择
类型多样，推动玩家选择流派/Build
2️⃣ 宝箱分类
类型	内容	特点
普通	常规武器/道具	数量多，价值低
精英	稀有武器/道具/抽卡	少量，价值高
事件/挑战	机制型道具或临时增益	玩家策略选择或达成条件获得
传奇	强力稀有装备	非常稀少，但可改变游戏局势
3️⃣ 宝箱掉落机制
每波/每关掉落概率：
普通 60%
精英 30%
传奇 10%
可触发“连锁抽卡”：开箱同时随机生成 1~2 张升级卡片
4️⃣ 宝箱设计原则
宝箱奖励应与玩家当前 Build / 流派产生联动
避免“白开局抽到极端强卡”破坏平衡
可设计“分支选择”：宝箱内2~3件物品，玩家只能选1件
四、系统循环整合图（Roguelike 核心循环）
玩家开始关卡 / 波数
打怪 → 获得金币/材料
击杀/触发事件 → 获得宝箱/抽卡机会
波数结束 → 进入商店
商店选择购买 → 武器/道具/临时buff
升级/升等级 → 免费抽卡（Perk）
回到下一波，循环

玩家在每一轮都必须做“选择”，同时 Build 可成长 → 上瘾感产生

五、核心设计注意事项
货币平衡
商店价格 + 宝箱掉落 + 升级抽卡数值必须调和
避免无限滚雪球导致游戏太容易或太难
保底机制
宝箱、抽卡或商店稀有物品有保底，避免玩家过度卡运气
流派引导
升级卡片 + 商店商品 + 宝箱奖励都可以偏向玩家当前 Build
但保留随机性，鼓励“临时混流派”
决策紧张感
玩家资源有限，每次选择都很关键（比如只能选1件道具或1张升级卡）
六、下一步优化方向
数值模拟
用你现有属性 + 武器 + 道具系统做一轮模拟，看看宝箱/商店/升级卡片的收益是否平衡
保底概率设计
保证每5波/每3次升级至少出现一次稀有卡/道具
流派驱动奖励
统计玩家当前 Build 标签，动态增加相关奖励出现概率
