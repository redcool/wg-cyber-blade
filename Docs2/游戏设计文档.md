# 游戏设计文档（GDD）
一、游戏概述

类型：

Roguelike + 自动射击（类《Brotato》）

核心玩法：

玩家控制角色在封闭场景中生存
自动攻击 + 手动移动
通过升级 / 商店 / 宝箱构建Build
二、核心循环（Game Loop）
战斗 → 掉落资源 → 升级 → 商店 → 强化 → 下一波 → Boss
三、核心系统
1️⃣ 战斗系统
自动攻击（基于武器）
玩家只控制移动
伤害公式：
FinalDamage = (Base + Flat) × (1 + Percent) × Crit × Special
2️⃣ 属性系统

分为：

生存（HP / Armor / Dodge）
输出（Damage / Crit / AttackSpeed）
功能（Speed / Range）
经济（Luck / Harvest）
3️⃣ 武器系统
类型：
Melee
Ranged
Magic
Tech
每个武器：
行为模式（子弹 / AOE / DOT）
Scaling规则
4️⃣ 道具系统
类型：
属性型
触发型
流派型
触发：
OnHit / OnKill / OnCrit
5️⃣ 流派系统（Build）

核心流派：

🔥 Fire（燃烧）
💥 Explosive（爆炸）
🗡️ Melee
🎯 Crit
🤖 Tech
💰 Economy
6️⃣ 敌人系统

敌人类型：

Chaser（追击）
Tank（高血）
Shooter（远程）
Swarm（群体）
Bomber（自爆）
7️⃣ 波次系统
每波有预算（Budget）
根据权重生成敌人
难度递增
8️⃣ Boss系统
多阶段（Phase）
不同技能组合
强制玩家移动
9️⃣ 奖励系统
商店
每波刷新
3~5个物品
升级
3选1卡片
宝箱
掉落装备 / 道具 / 卡片
四、数值设计原则
1️⃣ 成长曲线
前期：线性成长
中期：build形成
后期：指数爆发
2️⃣ 平衡原则
不允许无敌build
每个流派有克制关系
3️⃣ 软上限
Crit ≤ 60%
Dodge ≤ 50%
AttackSpeed ≤ 300%
